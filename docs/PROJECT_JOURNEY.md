## 1. Що це
**Mask Picker** — веб-інструмент (Flask + vanilla JS) для **ручної доразмітки**
датасету мікроскопії клітин (Cryobiology V). Пайплайн: ML-сегментація дає чорнові
маски кількома моделями → анотатор у браузері (1) **відбирає** найкращу модель і
**прибирає** погані маски (Cleanup), (2) **домальовує** пропущене полігонами
(Polygons), (3) **групує** клітини «1 ядро + N везикул» (Groups) → бекенд **запікає**
фінальний датасет (instance + semantic маски + групи) для замовника.

Мета датасету: ground-truth для алгоритму привʼязки везикул до ядер (clusterization).

## 3. Ключові складнощі та як вирішено (головна історія)
### 3.1. «Війна з десинхронізацією» (центральний виклик)
Найскладніше було НЕ написати фічі, а **втримати узгодженість даних**. Один логічний
концепт (напр. «відхилена клітина», «полігон у групі») мав **кілька представлень**
(збережене + обчислене, два ID-простори, паралельні UI-елементи), і консюмери
оновлювались **не всі разом** → виправляєш в одному місці, ламається в іншому. Серія
багів (Bug 3/4/5/6/7/13/14, double-count, «привид ядер», covered-rejected) — усі з цього кореня.

**Як вирішено (поетапно):**
- **Reserved-ID range (v1.16.0)** — фундамент: полігони отримують ID `50000+`, що
  ніколи не перетинаються з ID моделі → зникла ціла родина колізій. Уцілілі маски
  зберігають свій ID (`raw==baked`).
- **Layer 1/Layer 2 розмежування (v1.16.2)** — підрахунок ігнорує bake-артефакти,
  deliverable їх тримає → усунуто double-count і «привид».
- **Solution B (v1.16.2)** — інтерактивне групування рахується на фронті з робочих
  даних (завдяки reserved-ID), без 7-секундного запікання на кожну дію.
- **`CODE_AUDIT_PRINCIPLES.md`** — з болю народився метод: концепт-орієнтований
  аудит (знайди ВСІ представлення й консюмерів концепту, перш ніж правити).
- **Системний аудит + 14 arch-docs (01.06)** — трасування кожного звʼязку front↔back;
  підтверджено, що розбіжностей трактування в коді не лишилось; знайдено й закрито
  ще 5 глибших/крайових багів (F-001/004/007/008/011).

### 3.2. Інші складнощі
- **Дві «різні памʼяті»** (браузерний стан ефемерний vs файли на диску) — чітко
  розмежовано; autosave + flush + анти-race (generation-guard, проміс-черга).
- **Глобальний хронологічний undo** на 3 домени (замінив 3 окремі стеки — джерело розсинхрону).
- **Запозичений batch-експортер** (cellsegkit, Cryobiology III) використовувався в
  інтерактиві → Solution B вивів його з гарячого шляху (лишився для фінального deliverable).
- **Командна розмітка** (кілька анотаторів) — мердж результатів, єдиний формат.
- **Середовище:** Windows + кирилиця в шляхах, OneDrive, Flask threaded, відсутність GPU для частини моделей.
